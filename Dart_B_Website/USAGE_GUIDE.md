## 시작하기

### 백엔드 서버 실행
```bash
cd backend
python -m uvicorn main:app --reload --port 8000
```

### 프론트엔드 서버 실행
```bash
cd frontend
npm install
npm run dev
```

서버가 실행되면:
- 백엔드: http://localhost:8000
- 프론트엔드: http://localhost:5173

## 관리자 로그인

1. 웹사이트 접속 후 우측 상단의 **"관리자"** 버튼 클릭
2. 관리자 계정으로 로그인
   - 최초 로그인 시 관리자 계정이 필요합니다
   - 계정이 없다면 백엔드에서 직접 생성하거나 기존 계정을 사용하세요

## 멤버 관리

### 1. 개별 멤버 추가

1. 관리자 페이지에서 **"멤버 추가"** 버튼 클릭
2. 다음 정보 입력:
   - **이름** (필수)
   - **기수** (필수)
   - **전공**
   - **학번**
   - **이메일**
   - **직책** (예: "3기 기획팀장", "FND 교육팀장")
   - **GitHub**, **LinkedIn**, **Instagram** 링크
   - **프로필 이미지** 업로드
   - **임원 여부** 체크박스
   - **활성 멤버** 체크박스

3. **"추가"** 버튼 클릭

### 2. 멤버 수정

1. 멤버 목록에서 수정할 멤버의 **"수정"** 버튼 클릭
2. 정보 수정 후 **"수정"** 버튼 클릭

### 3. 멤버 삭제

1. 멤버 목록에서 삭제할 멤버의 **"삭제"** 버튼 클릭
2. 확인 메시지에서 **"확인"** 클릭

### 4. PDF 업로드로 일괄 추가

1. 관리자 페이지에서 **"PDF 업로드"** 버튼 클릭
2. 멤버 정보가 포함된 PDF 파일 선택
3. 업로드 완료 메시지 확인

## Google Forms 연동

Google Forms의 응답을 자동으로 멤버 데이터로 동기화하는 기능입니다.

### 사전 준비

#### 1. Google Sheets API 키 생성

1. [Google Cloud Console](https://console.cloud.google.com/) 접속
2. 프로젝트 선택 또는 새 프로젝트 생성
3. **"APIs & Services"** > **"Library"** 이동
4. **"Google Sheets API"** 검색 후 **"Enable"** 클릭
5. **"APIs & Services"** > **"Credentials"** 이동
6. **"+ CREATE CREDENTIALS"** > **"API key"** 선택
7. 생성된 API 키 복사

#### 2. Google Sheets API 키 제한 설정 (권장)

1. 생성한 API 키 클릭
2. **"API restrictions"** 섹션에서:
   - **"Restrict key"** 선택
   - **"Google Sheets API"** 선택
3. **"Save"** 클릭

#### 3. Google Forms와 Sheets 연결

1. Google Forms 열기
2. **"응답"** 탭 클릭
3. **"Google Sheets에 연결"** 버튼 클릭 (이미 연결되어 있으면 생략)
4. 새 시트 생성 또는 기존 시트 선택

#### 4. 시트 공유 설정

1. 생성된 Google Sheets 열기
2. 우측 상단 **"공유"** 버튼 클릭
3. **"링크가 있는 모든 사용자"** 또는 **"모든 사용자"** 선택
4. 최소 **"뷰어"** 권한 부여

#### 5. 시트 ID 확인

시트 URL에서 시트 ID 추출:
```
https://docs.google.com/spreadsheets/d/{시트_ID}/edit
```

예시:
- URL: `https://docs.google.com/spreadsheets/d/1k8CL_plPy4Y_OXYa_2ha6kqw5Sy1d5FujqE9EFc7MmI/edit`
- 시트 ID: `1k8CL_plPy4Y_OXYa_2ha6kqw5Sy1d5FujqE9EFc7MmI`

### Google Forms 동기화 실행

1. 관리자 페이지에서 **"구글 폼 동기화"** 버튼 클릭
2. 다이얼로그에서:
   - **구글 시트 ID** 입력 (시트 URL에서 복사)
   - **Google API 키** 입력
3. **"동기화 시작"** 버튼 클릭
4. 동기화 완료 메시지 확인

### Google Forms 필드 매핑

다음 필드명이 정확히 일치해야 합니다:

| Google Forms 필드명 | 데이터베이스 필드 |
|-------------------|----------------|
| 기수를 선택해주세요 | generation |
| 이름을 입력해주세요 | name |
| 전공을 학부형식으로 작성해주세요 | major |
| 운영진으로 활동한 경우 "n기 ~팀장" 형식으로 입력해주세요 | position, is_executive |
| (선택) 프로필에 올라갈 사진을 업로드해주세요 | profile_image_url |
| (선택) 링크드인 프로필 주소를 입력해주세요 | linkedin |
| (선택) 깃허브 주소를 입력해주세요 | github |
| (선택) 인스타그램 프로필 ID를 입력해주세요 | instagram |

### 주의사항

- Google Forms의 필드명이 정확히 일치해야 합니다
- 프로필 사진은 Google Drive 링크 형식이어야 합니다
- Google Drive 파일이 공개 설정되어 있어야 이미지 다운로드가 가능합니다
- 기존 멤버는 이름과 기수로 매칭되어 업데이트됩니다

## 이미지 관리

### 프로필 이미지 업로드

1. 멤버 추가/수정 시 **"프로필 이미지"** 섹션에서:
   - **"이미지 업로드"** 버튼 클릭
   - 이미지 파일 선택 (JPG, PNG, GIF, WEBP, SVG)
   - 최대 파일 크기: 10MB
2. 업로드된 이미지는 자동으로 미리보기 표시
3. 이미지 변경 시 **"이미지 변경"** 버튼 클릭
4. 이미지 삭제 시 **"X"** 버튼 클릭

### 로고 업로드

1. 관리자 페이지의 **"이미지 관리"** 섹션에서
2. **"로고 관리"** 카드에서 로고 업로드
3. 업로드된 로고는 웹사이트 헤더에 표시됩니다

### 일반 이미지 업로드

1. 관리자 페이지의 **"이미지 관리"** 섹션에서
2. **"일반 이미지"** 카드에서 이미지 업로드
3. 업로드된 이미지는 갤러리에서 확인 가능

## Members 페이지 사용법

### 기수 선택

1. Members 페이지 상단의 **기수 선택 드롭다운** 클릭
2. 다음 옵션 중 선택:
   - **전체**: 모든 기수의 멤버 표시
   - **1기, 2기, 3기...**: 특정 기수의 멤버만 표시

### 필터링

1. **"전체"** 버튼: 모든 멤버 표시
2. **"Executives"** 버튼: 임원만 표시
3. **"Members"** 버튼: 일반 멤버만 표시

### 정렬 순서

- **기수 순**: 1기부터 순서대로 표시
- **같은 기수 내**:
  - Executives: FND 팀장이 먼저, 그 다음 숫자 기수 팀장, 같은 우선순위면 이름순
  - Members: 이름순 (가나다순)

### 멤버 정보 확인

각 멤버 카드에서 확인 가능한 정보:
- 프로필 사진
- 이름
- 전공
- 직책 (Executives만)
- 연락처 아이콘 (이메일, GitHub, LinkedIn, Instagram)

### 연락처 사용

- **이메일**: 클릭 시 클립보드에 복사
- **GitHub, LinkedIn, Instagram**: 클릭 시 새 탭에서 프로필 열기

## 문제 해결

### Google Forms 동기화 오류

**404 에러 (시트를 찾을 수 없음)**
- 시트 ID가 올바른지 확인
- 시트가 공유되어 있는지 확인
- 시트 URL에서 `/d/`와 `/edit` 사이의 전체 문자열을 복사했는지 확인

**403 에러 (접근 권한 없음)**
- API 키가 올바른지 확인
- Google Sheets API가 활성화되어 있는지 확인
- 시트 공유 설정 확인

**400 에러 (잘못된 요청)**
- 시트에 데이터가 있는지 확인
- 필드명이 정확히 일치하는지 확인

### 이미지가 표시되지 않음

1. 브라우저 콘솔에서 에러 메시지 확인
2. 이미지 URL이 올바른지 확인
3. 백엔드 서버가 실행 중인지 확인
4. 이미지 파일이 `uploads/members/` 디렉토리에 있는지 확인

### 멤버가 표시되지 않음

1. 멤버의 **"활성 멤버"** 체크박스가 체크되어 있는지 확인
2. 선택한 기수에 해당 멤버가 있는지 확인
3. 필터 설정 확인 (전체/Executives/Members)
